1. Ce este o resursă în aplicația ta?
O resursă este orice spațiu sau serviciu al campusului despre care studenții pot obține informații: locație, program de funcționare, capacitate și categorii. Exemple: Biblioteca Centrală, Cantina Principală, Aula Magna.

2. Dă exemplu de un URI și explică componentele:
https://campus-info-hub.example.com/pages/library.html#schedule
Schemă: https
Host: campus-info-hub.example.com
Path: /pages/library.html
Fragment: #schedule

3. Care părți sunt statice și care dinamice?
Statice: fișierele HTML, CSS, JSON
Dinamice: lista de resurse generată de JS, filtrarea pe tag-uri, secțiunea "Locuri de studiu"

4. Este aplicația document-centric sau interactivă?
Ambele — paginile individuale sunt document-centric, iar homepage-ul e interactiv prin fetch + filtrare.
